## Why

OpenSpec users need a chronological view of what changed, what validated, and what moved to archive, especially across active local work.

## What Changes

- Add a timeline/activity view derived from local file metadata, git history, validation runs, and archive state.
- Let users filter activity by repo, change, capability, and event type.

## Impact

- Activity is derived and rebuildable, not canonical app state.
- Git integration remains read-only for timeline context.
