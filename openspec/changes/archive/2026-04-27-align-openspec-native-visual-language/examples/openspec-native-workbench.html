Align OpenSpec Native Visual Language

openspec/changes/align-openspec-native-visual-language

Checked
Proposal Design Tasks Spec changes
proposal.md

Why

OpenSpec Studio should feel like a native companion to OpenSpec rather than a generic dashboard wrapped around OpenSpec data.

What Changes

  • Artifact-first layout with source paths and markdown previews up front.
  • Quiet status pills that read as trust cues, not dashboard badges.
  • Document and code surfaces styled like OpenSpec examples.
+ Requirement: OpenSpec-native visual language
The system SHALL express a restrained, text-first visual language.
- Dashboard-style noise and repeated status language
Source text remains the center of review.
