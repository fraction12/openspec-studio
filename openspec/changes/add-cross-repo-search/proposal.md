## Why

Users need to find related OpenSpec work across proposals, designs, tasks, specs, and validation output without manually searching folders.

## What Changes

- Add search across indexed OpenSpec artifacts.
- Support repo, change, spec, artifact type, status, and text filters.
- Keep results derived from local files and validation output.

## Impact

- Search index derived from current in-memory repo indexes.
- No hosted search or app-private canonical content store.
